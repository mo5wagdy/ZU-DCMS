## 🚀 The Story

In most university dental clinics, the system isn't really a "system".

Patients wait for hours.  
Students struggle to complete their requirements.  
Intern doctors manually distribute cases.  
Admins rely on spreadsheets… and hope.

Everything works — but nothing flows.

That's where **ZU-DCMS** was born.

- Patients

…into one seamless, integrated ecosystem.

## 🧠 What Makes It Different?

This system doesn't just store data — it **thinks**.

### ⚖️ Smart Case Distribution
An AI-powered prioritization system ensures:
- Students with **0 cases** get priority
- Fair and balanced workload distribution
- No student gets left behind
- Optimal case assignment based on student expertise

---

### 🧩 Clean Architecture, Real Engineering
Built using **Clean Architecture** principles:
- **Domain Layer** — Core business entities and rules
- **Application Layer** — Use cases and business logic via CQRS
- **Infrastructure Layer** — Database, external services, and persistence
- **API Layer** — Controllers, endpoints, and HTTP interfaces
- Fully decoupled layers
- Scalable, testable, maintainable
- Repository Pattern + Unit of Work
- DTOs + Validation Layer
- Global Exception Handling

Because this isn't a demo project…  
it's built for **production reality**.

---

### 📊 Full System Control
Admins can control everything:
- Daily patient limits
- Session timing
- Pricing and fees
- Academic terms and requirements
- Student capabilities and qualifications
- System-wide configurations

All configurable. No hardcoding.

## 🏗️ Tech Stack

### Backend
- **Backend:** .NET Web API (Clean Architecture)
- **Database:** SQL Server
- **Command Query Separation:** CQRS with MediatR
- **Architecture:** Clean Architecture with SOLID Principles
- **Patterns:** Repository Pattern, Unit of Work, Dependency Injection

### Frontend
- **Repository:** [ZU-DCMS-REACT](https://github.com/mo5wagdy/ZU-DCMS-React)
- **Framework:** React with modern TypeScript
- **UI/UX:** Responsive design with intuitive patient, student, and admin interfaces
- **State Management:** Efficient global state handling
- **API Integration:** Seamless backend communication
