# 🏥 UD-CMS — University Dental Clinic Management System

> From chaos… to clarity.  
> From manual struggle… to intelligent orchestration.

---

## 🚀 The Story

In most university dental clinics, the system isn’t really a “system”.

Patients wait for hours.  
Students struggle to complete their requirements.  
Intern doctors manually distribute cases.  
Admins rely on spreadsheets… and hope.

Everything works — but nothing flows.

That’s where **UD-CMS** was born.

---

## 💡 The Vision

UD-CMS is not just a management system.

It’s a **smart orchestration engine** that connects:
- Patients
- Students
- Intern Doctors
- Admins

…into one seamless, real-time ecosystem.

---

## 🧠 What Makes It Different?

This system doesn’t just store data — it **thinks**.

### ⚖️ Smart Case Distribution
An AI-powered prioritization system ensures:
- Students with **0 cases** get priority
- Fair and balanced workload distribution
- No student gets left behind

---

### ⚡ Real-Time Everything
With SignalR powering the core:
- Live dashboard updates
- Instant notifications
- No refresh. No delays.

---

### 🧩 Clean Architecture, Real Engineering
Built using **Clean Architecture** principles:
- Domain-driven design
- Fully decoupled layers
- Scalable, testable, maintainable

Because this isn’t a demo project…  
it’s built for **production reality**.

---

### 💳 Seamless Payment Integration
Integrated with **Fawry**:
- Generate payment codes instantly
- Track payment status
- Reliable fallback strategies

---

### 📊 Full System Control
Admins can control everything:
- Daily patient limits
- Session timing
- Pricing
- Academic terms
- Student requirements

All configurable. No hardcoding.

---

## 🔄 The Flow (In Real Life)

1. A patient books an appointment  
2. Gets notified instantly  
3. Intern doctor diagnoses and assigns a case  
4. AI suggests the best student  
5. Student handles the case across sessions  
6. Progress is tracked automatically  
7. Admin sees everything — live

No paper. No confusion. No chaos.

---

## 🏗️ Tech Stack

- **Backend:** .NET Web API (Clean Architecture)
- **Database:** SQL Server
- **Realtime:** SignalR
- **Payments:** Fawry Integration
- **AI Layer:** External AI Agent
- **Notifications:** SMS, WhatsApp, Email
